1. PROJECT OVERVIEW
================================================================================

DESCRIPTION:
A comprehensive property management system enabling owners, property managers,
and tenants to manage rental properties, leases, payments, and maintenance
requests.

USER ROLES:
- Owner: Manages properties, assigns managers, views all data
- Property Manager: Manages assigned properties, units, tenants
- Tenant: Views lease info, makes payments, submits maintenance requests

KEY FEATURES:
- Multi-role authentication with Supabase Auth
- Property and unit management
- Lease lifecycle management
- Payment tracking with overdue detection
- Maintenance request system with image uploads
- Document storage and management
- Role-based dashboards with analytics

12.2 DATA FLOW
-------------
Client → Next.js Server Components → Supabase Client → PostgreSQL
                                  ↓
                            Row Level Security (RLS)
                                  ↓
                           Authorized Data Only

12.3 STATE MANAGEMENT
--------------------
- Server Components for data fetching (default)
- React Server Actions for mutations
- Local state with useState for UI
- URL state for filters/pagination
- Context API for global user state

12.4 KEY UTILITIES
-----------------
□ Date formatting (date-fns):
  - formatDate(date, format)
  - calculateDaysBetween(start, end)
  - isOverdue(dueDate)

□ Currency formatting:
  - formatCurrency(amount)
  - parseCurrency(string)

□ File handling:
  - uploadFile(file, bucket, path)
  - getPublicUrl(path)
  - deleteFile(path)

□ Validation schemas (Zod):
  - propertySchema
  - unitSchema
  - leaseSchema
  - tenantSchema
  - paymentSchema
  - maintenanceSchema

13. SECURITY CONSIDERATIONS
================================================================================

13.1 AUTHENTICATION SECURITY
----------------------------
✓ Supabase Auth with RLS policies
□ Implement rate limiting for login attempts
□ Strong password requirements
□ Email verification for new accounts
□ Secure session management with httpOnly cookies
□ CSRF protection (Next.js built-in)

13.2 AUTHORIZATION
-----------------
✓ Row Level Security policies on all tables
□ Server-side permission checks before mutations
□ Role-based UI rendering
□ API route protection with middleware
□ Verify user has access to resources

13.3 DATA SECURITY
-----------------
□ Input validation on client and server
□ SQL injection prevention (using Supabase client)
□ XSS prevention (React auto-escaping)
□ File upload validation:
  - File type whitelist
  - File size limits
  - Malware scanning (if budget allows)
  - Unique filenames to prevent overwrites

13.4 PRIVACY
-----------
□ HTTPS only (enforced by Vercel)
□ Sensitive data encryption at rest (Supabase)
□ Personal data handling compliance
□ Secure document storage with signed URLs
□ Audit logs for sensitive operations
